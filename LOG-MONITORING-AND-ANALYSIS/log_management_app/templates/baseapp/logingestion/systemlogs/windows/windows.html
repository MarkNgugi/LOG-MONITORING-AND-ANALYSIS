3. Running the Script on Windows

Provide detailed instructions on how to run the downloaded script on the user's Windows machine and set it up as a service to ensure it starts on boot.

  1. Download the script using the button above.
  2. Open Command Prompt as Administrator.
  3. Navigate to the location where the script was downloaded.
  4. Run the script with the following command: powershell -ExecutionPolicy Bypass -File setup-log-collection.ps1
  5. Ensure the script runs successfully.
  6. To set up the script as a service, follow these steps:
    1. **Create a Wrapper Batch File:**
      • Create a new batch file in the same directory as the PowerShell script or another appropriate location (e.g., `C:\Scripts\log-collection-service.bat`).
      • Add the following content to the batch file:
        • @echo off
                                    powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Scripts\setup-log-collection.ps1"
    2. **Create the Service Using `sc.exe`:**
      • In the Command Prompt (still running as Administrator), run the following command:
        • sc create LogCollectionService binPath= "C:\Scripts\log-collection-service.bat" start= auto
      • This command creates a new service named `LogCollectionService` that will run the batch file at startup.
    3. **Configure Service Properties (Optional):**
      • Press `Win + R`, type `services.msc`, and press Enter to open the Services management console.
      • Find **LogCollectionService** in the list, right-click, and select **Properties**.
      • Adjust the **Startup type** and **Recovery** options as needed.
    4. **Start the Service:**
      • Start the service by running the following command in the Command Prompt:
        • net start LogCollectionService
    5. **Verify the Service is Running:**
      • Ensure the service is running correctly by checking the **Services** management console or running the following command:
        • sc query LogCollectionService
