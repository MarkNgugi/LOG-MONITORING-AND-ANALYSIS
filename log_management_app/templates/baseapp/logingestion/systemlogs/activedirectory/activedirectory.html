About this integration

The integration of Linux server monitoring for system logs and authentication logs is a critical component for ensuring server security and operational integrity. By leveraging logs from files such as /var/log/syslog and /var/log/auth.log, administrators can implement proactive alert systems that detect specific anomalies and potential threats. These logs provide granular details about system activities, including authentication attempts, service restarts, system performance, and user actions, allowing for real-time monitoring and response.

The integration highlights the importance of identifying specific events, such as multiple failed SSH login attempts, unauthorized root access, and kernel out-of-memory (OOM) killer activations. By categorizing these alerts into authentication and system-level events with varying severity levels, administrators can prioritize responses and mitigate risks. Furthermore, automated monitoring solutions, combined with this granular log analysis, enable businesses to adhere to compliance standards, reduce downtime, and enhance overall server resilience. This approach ensures that the Linux server remains secure and operates efficiently in the face of both internal and external challenges.

2. Create a Systemd Service File

Systemd services are configured using .service files. Follow these steps to create a service file for your script:

  1. Create a service file using the command below:
    2. sudo nano /etc/systemd/system/log-collector.service
  2. Add the following content to the service file:
    3. [Unit]
Description=Log Collector Service
After=network.target
            
[Service]
ExecStart=/usr/local/bin/collect-logs.sh
Restart=always
RestartSec=5s
User=root
Environment=LOG_API_URL="https://example.com/api/logs"
            
[Install]
WantedBy=multi-user.target
  3. Save and close the file.

3. Reload Systemd and Enable the Service

Reload the systemd configuration to recognize the new service and enable it to start on boot.

  1. Reload the systemd configuration using the command below:
    2. sudo systemctl daemon-reload
  2. Enable the service to start on boot:
    3. sudo systemctl enable log-collector.service
  3. Start the service using the command below:
    4. sudo systemctl start log-collector.service
  4. Check the service status to ensure it is running:
    5. sudo systemctl status log-collector.service

Alerts & Anomalies

Alert Name Type Category Level Description
Multiple Failed Login Attempts Across Domain Alert Authentication High More than 5 failed login attempts across multiple domain controllers within a short period, indicating a potential brute-force attack.
Successful Domain Admin Login from Unusual IP Alert Authentication Critical Domain administrator login from an IP address outside the normal range, possibly indicating a compromised account.
Creation of Unusual User Accounts Alert Authentication Medium New user accounts created with elevated privileges or from unknown sources, indicating potential unauthorized access.
Admin Group Membership Changes Alert Authentication Critical Changes to the membership of sensitive groups like Domain Admins, Enterprise Admins, or Administrators, indicating possible privilege escalation.
Unexpected Domain Controller Failover Alert System High Unexpected failover of a domain controller, which could indicate a problem with the system or potential attack.
Domain Trust Relationship Failure Alert System High Failure of a domain trust relationship between domain controllers, which could indicate tampering with domain security settings.
Account Lockout Across Multiple Domains Alert Authentication High Multiple account lockouts across different domains in a short period, potentially indicating a brute force or password spraying attack.
Unexpected Changes to Group Policies Alert Authentication Critical Unexpected or unauthorized changes to Active Directory Group Policy Objects (GPOs), which could allow privilege escalation or bypass security controls.
Domain Admin Account Password Change Alert Authentication High Change to the password of a Domain Admin account, which could indicate an attempt to hijack or compromise an administrative account.
Failed Authentication Attempts for Domain Admin Accounts Alert Authentication Critical Multiple failed login attempts for Domain Admin accounts, indicating a potential brute force or targeted attack on privileged accounts.
Unusual SID History Modifications Alert Authentication Critical Changes to the Security Identifier (SID) history of user or computer accounts, which could indicate an attempt to escalate privileges or bypass security.
Changes to Service Principal Names (SPNs) Alert Authentication High Changes to Service Principal Names (SPNs), indicating potential Kerberos ticket manipulation or attempts to impersonate services.
Unusual Account Logon Locations Alert Authentication Medium Logon attempts from unusual geographic locations or IP addresses for user accounts, indicating possible credential theft or misuse.
Multiple Password Reset Requests Alert Authentication High Multiple failed or successful password reset requests for the same account, indicating potential compromise or social engineering.
Unusual Service Account Behavior Alert Authentication Medium Service accounts performing actions that are outside their usual pattern of behavior, potentially indicating misuse or compromise.
Unregistered or Unrecognized Domain Controllers Alert System Critical Unregistered or rogue domain controllers attempting to join the Active Directory domain, indicating potential attack or misconfiguration.
LDAP Injection Attempts Alert Authentication High Unusual patterns of LDAP queries or failed authentication attempts, possibly indicating an LDAP injection attack on the AD infrastructure.
